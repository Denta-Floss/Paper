# Pixel Perfect Checklist

Run this checklist before finalizing a Flutter UI implementation.

## Structure

- Screen sections match design hierarchy.
- Scrolling behavior matches design intent.
- Navigation and app chrome placement are correct.

## Spacing and Sizing

- Outer margins match spec.
- Internal paddings match spec.
- Inter-element gaps are consistent.
- Component widths and heights are accurate.

## Typography

- Font family matches design system.
- Font sizes and weights are correct by hierarchy level.
- Line-height and letter spacing are acceptable for the target design.
- Truncation and wrapping behavior align with expected UX.

## Visual Styling

- Colors match tokens or sampled values.
- Border radius values match component style.
- Borders and dividers are consistent in weight and opacity.
- Shadows and elevation are visually equivalent.

## Interaction and States

- Press, hover, focus, selected, and disabled states are styled correctly where relevant.
- Buttons and inputs expose clear state feedback.

## Responsive Behavior

- Layout adapts cleanly across phone and tablet widths.
- Critical content never overflows or clips unexpectedly.
- Grids and rows reflow at defined breakpoints.

## Accessibility

- Icon-only controls have semantic labels.
- Important controls have adequate touch targets.
- Contrast remains readable in all key surfaces.
